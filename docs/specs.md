# Sky Quality Analyzer 技術仕様書

## 1. SQMとは何か

### 1.1 空の明るさの指標としての SQM

SQM（Sky Quality Meter）は、夜空の明るさを定量的に表す指標です。Unihedron 社が開発したハンドヘルドデバイス「Sky Quality Meter」に由来しており、現在では測定値そのものを指す用語として広く使われています。

SQM 値の単位は **mag/arcsec²（等級/角秒²）** です。天文学の「等級（magnitude）」は対数スケールで定義されているため、**SQM 値が大きい（＝数値が大きい）ほど空が暗い**という、直感とは逆の関係になります。

```
SQM 22.0  →  暗い空（良好）
SQM 18.0  →  明るい空（光害あり）
```

### 1.2 単位 mag/arcsec² の意味

`mag/arcsec²` は「1角秒²あたりの面輝度を等級で表したもの」です。

- **mag（等級）**: 天体の明るさを表す対数スケール。5等級の差がフラックス比 100 倍に相当
- **arcsec²（角秒²）**: 天球上の面積の単位。1角秒 = 1/3600 度 = 約 4.85 × 10⁻⁶ ラジアン

つまり SQM 値とは、「夜空の 1角秒²の明るさが何等級か」を表します。空全体を均一な面光源とみなし、その面輝度を測定しています。

### 1.3 典型的な値と空の状態

| SQM 値（mag/arcsec²） | 空の状態 |
|----------------------|---------|
| 22.0 以上             | 完璧な暗黒空。天の川の中心部が地上に影を作るほど |
| 21.5 〜 22.0          | 真の暗黒空。微光天体の観測が可能 |
| 21.0 〜 21.5          | 優良。農村地帯の空 |
| 20.0 〜 21.0          | 良好。郊外の空 |
| 19.0 〜 20.0          | 普通。郊外と都市の中間 |
| 18.0 〜 19.0          | 光害あり。明るい郊外 |
| 18.0 未満             | 都市部。天の川はほぼ見えない |

### 1.4 ボルトル暗天スケールとの対応

ボルトル（Bortle）暗天スケールは、夜空の暗さを 1〜9 のクラスで表す指標です。SQM 値との対応はおおよそ以下のとおりです。

| ボルトルクラス | SQM（mag/arcsec²） | 特徴 |
|-------------|-------------------|------|
| 1           | 22.0 以上          | 完璧な暗黒空 |
| 2           | 21.5 〜 22.0       | 真の暗黒空 |
| 3           | 21.3 〜 21.5       | 農村の空 |
| 4           | 20.4 〜 21.3       | 農村・郊外の移行帯 |
| 5           | 19.1 〜 20.4       | 郊外の空 |
| 6           | 18.0 〜 19.1       | 明るい郊外 |
| 7           | 17.5 〜 18.0       | 郊外・都市の移行帯 |
| 8 〜 9      | 17.5 未満          | 都市部 |

ボルトルスケールは観測者の主観も含むため、SQM 値のほうが定量的・再現性のある指標として科学的に有用です。

### 1.5 なぜ天文観測にとって重要か

SQM 値は以下の用途で活用されます。

- **撮影条件の記録**: 同じ機材・対象でも、SQM 値が異なれば比較できない。撮影ログに SQM を残すことで客観的な条件管理が可能
- **画像処理の基準**: バックグラウンドのノイズ特性は SQM に依存する。処理パラメータの参考値として利用
- **遠征先の評価**: 候補地の空質を定量的に比較・蓄積できる
- **光害マップへの貢献**: 個人の観測データを Globe at Night などのプロジェクトに提供し、光害の長期変化を追跡

## 2. 夜空の輝度の構成要素

SQM が測定するのは「夜空全体の明るさ」です。これは複数の異なる発光・散乱成分の合計です。各成分を理解することは、SQM 値を正しく解釈するうえで重要です。

### 2.1 光害（人工光の散乱）

都市・住宅・道路の照明が大気中の分子・エアロゾルで散乱したもの。SQM に最も大きな影響を与える成分です。

- 波長は光源の種類に依存（LED 照明が増加傾向）
- 大気の透明度（湿度・塵）によって散乱量が変化
- 地平線付近ほど強く、天頂方向は相対的に弱い

### 2.2 大気光（Airglow）

地球大気上層（高度 85 〜 300 km）での化学発光。光害のない場所でも SQM の下限を決める自然の成分です。

- 主な発光: OI（557.7 nm の緑、630 nm の赤）、OH バンド（近赤外）
- 天頂 SQM に対して約 1〜2 mag/arcsec² 分の影響
- 太陽活動サイクルや季節によって変化

### 2.3 黄道光・対日照

太陽系内の塵が太陽光を散乱したもの。春の夕方や秋の明け方に黄道に沿って見られる淡い光です。

- 黄道面に沿って広がる（黄道光）、地球の正反対方向に集中（対日照）
- 天頂 SQM への影響は小さいが、低緯度の観測地では考慮が必要

### 2.4 天の川・銀河背景光

分解できない無数の恒星が重なった拡散光。視線方向（銀緯）によって大きく異なります。

- 銀河中心方向（いて座付近）では最も明るく、SQM を数 tenths mag/arcsec² 低下させる
- 銀緯が高い方向では影響が小さい

### 2.5 まとめ

画像から求めた SQM 値は、これらすべての成分を含む **「その方向の空の合計面輝度」** です。「光害のない場所でも SQM が 22.0 を大きく超えない」のは、大気光・黄道光・銀河背景光が自然の下限を作っているためです。

## 3. 従来の SQM 測定方法

### 3.1 Unihedron SQM デバイスの仕組み

Unihedron 社の Sky Quality Meter は、光-周波数変換センサー（TAOS TSL237S）と Hoya CM-500 レンズを組み合わせたデバイスです。

- **視野角**: 約 80° FWHM（広角）
- **分光感度**: ピーク約 500〜600 nm（視感度に近い V バンド相当）
- **測定範囲**: 10 〜 22 mag/arcsec²
- **精度**: ±0.1 mag/arcsec²

センサーに入射した光量を周波数（Hz）に変換し、ファームウェアが SQM 値に変換して出力します。SQM-L（レンズ付き、狭視野版）と SQM-LE（イーサネット接続版）もあります。

### 3.2 デバイス測定の限界

| 制約 | 内容 |
|------|------|
| 広視野平均 | 80° 視野内の平均値のため、局所的な空の情報は得られない |
| 分光情報なし | 単一バンドの測定のため、光害の種類（LED vs. 水銀灯）を区別できない |
| 絶対較正 | 個体差・経年変化による誤差が生じる場合がある |
| 記録の手間 | 撮影と同時に自動記録するには追加のハードウェアが必要 |

### 3.3 画像から算出するアプローチとの比較

| 項目 | SQM デバイス | 画像から算出（本ツール） |
|------|-------------|------------------------|
| 追加ハードウェア | 必要 | 不要（撮影画像のみ） |
| 視野 | 広視野平均（80°） | 撮影視野内（フィルター・方向特定可能） |
| 分光情報 | なし（V バンド相当のみ） | フィルターに対応 |
| 時刻記録 | 自動 | FITS ヘッダー（DATE-OBS）から取得 |
| 精度 | ±0.1 mag/arcsec²（高精度） | ゼロ点較正の質に依存（±0.2〜0.5 程度） |
| 後処理での利用 | できない（リアルタイム測定のみ） | 既存の撮影データから後処理で算出可能 |

画像からの算出はデバイスより精度が劣る場合がありますが、**追加機材なしに既存の撮影データを活用できる**点が最大のメリットです。

## 4. 画像から SQM を算出する原理

### 4.1 測光の基礎

天文測光では、天体の明るさを「等級（magnitude）」で表します。等級の定義は以下のとおりです。

```
m = -2.5 × log₁₀(F) + C
```

- `F`: 検出器に入射したフラックス（光子数/秒 など）
- `C`: ゼロ点定数（測光系によって決まる）

等級は **対数スケール** であり、5等級の差がフラックス比 100 倍に対応します。また、**等級が小さいほど明るい**（負の値ほど非常に明るい）という点に注意が必要です。

### 4.2 面輝度（mag/arcsec²）の概念

点源（星）の明るさは「等級」で表しますが、空のバックグラウンドや系外銀河など広がった天体の明るさは **面輝度（Surface Brightness）** で表します。

面輝度は「単位立体角あたりの明るさを等級で表したもの」であり、単位は mag/arcsec²（等級/角秒²）です。

等級と面輝度の関係:
```
μ = m_pixel + 2.5 × log₁₀(Ω)
```

- `μ`: 面輝度 [mag/arcsec²]
- `m_pixel`: 1ピクセルの等級
- `Ω`: 1ピクセルがカバーする立体角 [arcsec²]

1ピクセルが `s` arcsec/px のピクセルスケールを持つ場合、`Ω = s²` [arcsec²/px] です。

### 4.3 バックグラウンド輝度の測定

撮影画像のピクセル値（ADU）には、以下の成分が含まれています。

```
ADU_total = ADU_sky + ADU_bias + ADU_dark + ADU_stars
```

- `ADU_sky`: 空のバックグラウンド（求めたい成分）
- `ADU_bias`: バイアス（読み出しオフセット）
- `ADU_dark`: ダーク電流
- `ADU_stars`: 星からの寄与

キャリブレーション済みの画像（バイアス・ダーク・フラット補正済み）では、バイアスとダーク電流は除去されています。さらに星を除外した領域の中央値または平均値を取ることで、`ADU_sky` を推定します。

### 4.4 ADU から面輝度への変換の考え方

キャリブレーション済み画像の空バックグラウンド `ADU_sky` [ADU/px] から面輝度（SQM 値）を算出する流れ:

1. **ADU を電子数に変換**: `e⁻ = ADU × Gain` [e⁻/px]
2. **露出時間で割る**: `flux = e⁻ / t_exp` [e⁻/s/px]
3. **等級に変換（ゼロ点使用）**: `m_pixel = -2.5 × log₁₀(flux) + ZP`
4. **ピクセルスケールで面輝度に変換**: `μ = m_pixel + 2.5 × log₁₀(pixel_scale²)`

この `μ` が SQM 値に相当する面輝度です。

## 5. 算出に必要なパラメータ

### 5.1 パラメータ一覧

| パラメータ | 単位 | 取得元 | 説明 |
|-----------|------|--------|------|
| `ADU_sky` | ADU/px | 画像解析 | バックグラウンド輝度（中央値） |
| `t_exp` | 秒 | FITS: `EXPTIME` | 露出時間 |
| `pixel_scale` | arcsec/px | FITS: `PIXSCALE` または計算 | ピクセルスケール |
| `Gain` | e⁻/ADU | FITS: `GAIN` またはカメラ仕様 | カメラゲイン |
| `ZP` | mag | 測光較正 | ゼロ点等級 |
| `Filter` | — | FITS: `FILTER` | 使用フィルター |
| `Binning` | — | FITS: `XBINNING` | ビニング（ピクセルスケール補正に必要） |

### 5.2 露出時間（EXPTIME）

FITS ヘッダーの `EXPTIME` または `EXPOSURE` キーワードから取得します。単位は秒。露出時間が長いほど同じ空でも ADU が大きくなるため、正規化に必須のパラメータです。

### 5.3 ピクセルスケール（pixel_scale）

ピクセルスケールは 1 ピクセルが対応する天球上の角度（arcsec/px）です。

FITS ヘッダーに `PIXSCALE` がある場合はそれを使用します。ない場合は以下の式で計算できます。

```
pixel_scale = (pixel_size_μm / focal_length_mm) × 206.265
```

- `pixel_size_μm`: カメラのピクセルサイズ [μm]
- `focal_length_mm`: 鏡筒の焦点距離 [mm]
- `206.265`: 変換係数（ラジアン→角秒）

プレートソルブ済みの画像では WCS キーワード（`CD1_1`, `CD1_2` など）から高精度なピクセルスケールを導出できます。

### 5.4 ゲイン（Gain）

ゲインは 1 ADU が何電子（e⁻）に対応するかを表す変換係数です（単位: e⁻/ADU）。

FITS ヘッダーの `GAIN` キーワードから取得するか、カメラのマニュアルを参照します。ゲインが不明な場合、ゼロ点較正をADUベースで行うことでゲインを陽に使わずに算出できます（後述）。

### 5.5 ゼロ点等級（ZP）

ゼロ点（Zero Point）は測光系を絶対等級スケールに結びつける定数です。

```
m_true = -2.5 × log₁₀(ADU / t_exp) + ZP
```

ゼロ点の求め方（精度の高い順）:

1. **カタログ較正**: 画像内の既知の星のカタログ等級と測定値を比較して算出（最も精度が高い）
2. **Astrometry.net / Siril の出力**: プレートソルブや測光較正を行ったソフトウェアが FITS ヘッダーに書き込む場合がある（`MAGZERO`, `MAGZP` など）
3. **経験値**: 機材・フィルター・大気条件の典型値を手動入力（最も簡便だが精度は低い）

### 5.6 フィルター

使用フィルターによって測定する波長域が異なるため、SQM 値の解釈が変わります。

| フィルター | 中心波長 | SQM との関係 |
|-----------|---------|-------------|
| L（ルミナンス）| 全可視域 | Unihedron SQM に近い（V バンド相当） |
| V（緑） | 550 nm | SQM デバイスの分光感度に最も近い |
| R（赤） | 640 nm | 光害（LED・ナトリウム灯）に敏感 |
| Ha（Hα） | 656 nm | 輝線フィルター。光害の影響を大幅にカット |
| B（青） | 450 nm | 大気散乱の影響が大きい |

フィルターなし（OSC/DSLR の RGB 合成）の場合は実効波長が不明確になるため、別途考慮が必要です。

## 6. SQM 算出の数学

### 6.1 バックグラウンド推定アルゴリズム

天文画像のバックグラウンドには星・銀河などの点源・拡散源が含まれるため、単純な平均では過大評価になります。以下のアルゴリズムで空のバックグラウンドレベル `ADU_sky` を推定します。

**シグマクリッピング統計**:

1. 画像全体（または指定領域）のヒストグラムを作成
2. 中央値（median）と標準偏差（σ）を計算
3. 中央値 ± n×σ（通常 n=3）の範囲外の値（星の寄与）を除外
4. 残ったピクセルの中央値を `ADU_sky` とする

中央値を使う理由: 平均値は輝点（星）の影響を強く受けるが、中央値は外れ値に頑健です。

### 6.2 ADU から面輝度への変換式

以下がメインの変換式です。

**ゲインを使う場合（電子数ベース）:**

```
μ = -2.5 × log₁₀(ADU_sky × Gain / t_exp) + ZP_e + 5 × log₁₀(pixel_scale)
```

- `ZP_e`: 電子数ベースのゼロ点 [mag]

**ADU ベースで計算する場合（ゲイン不要）:**

```
μ = -2.5 × log₁₀(ADU_sky / t_exp) + ZP_ADU + 5 × log₁₀(pixel_scale)
```

- `ZP_ADU = ZP_e + 2.5 × log₁₀(Gain)`
- ゼロ点 `ZP_ADU` にゲイン分が吸収されるため、ゲインを別途入力しなくてよい

この式の導出:

```
1. ピクセル 1 個のフラックス [ADU/s]:
   f = ADU_sky / t_exp

2. ピクセル等級:
   m_pixel = -2.5 × log₁₀(f) + ZP_ADU
           = -2.5 × log₁₀(ADU_sky / t_exp) + ZP_ADU

3. 1 ピクセルのカバーする立体角 [arcsec²]:
   Ω = pixel_scale²

4. 面輝度（mag/arcsec²）への変換:
   μ = m_pixel + 2.5 × log₁₀(Ω)
     = m_pixel + 2.5 × log₁₀(pixel_scale²)
     = m_pixel + 5 × log₁₀(pixel_scale)

5. まとめ:
   μ = -2.5 × log₁₀(ADU_sky / t_exp) + ZP_ADU + 5 × log₁₀(pixel_scale)
```

### 6.3 ゼロ点較正の方法

#### カタログ較正によるゼロ点の算出

画像内の恒星をカタログ等級と照合し、ゼロ点を実測値から決定します。

```
ZP_ADU = m_catalog + 2.5 × log₁₀(ADU_star / t_exp)
```

複数の星について算出した値の中央値をゼロ点とします。大気の消光補正を行う場合は:

```
m_true = m_observed + k × X
```

- `k`: 消光係数（V バンドで典型値 0.1〜0.2 mag/気団）
- `X`: 大気質量（Air Mass）= `1 / cos(天頂角)` （近似）

高精度が必要な場合は大気消光補正が必要ですが、SQM の相対的な比較には省略可能です。

### 6.4 ビニングの補正

ビニング `n × n` で撮影した画像では、1ピクセルが n² 個の元ピクセルに相当します。

```
pixel_scale_binned = pixel_scale_1x1 × n
```

FITS ヘッダーの `XBINNING` / `YBINNING` から取得し、ピクセルスケールに乗算して補正します。

## 7. 実装仕様

_（設計中 — 以下はドラフト）_

### 7.1 アルゴリズム概要

```
入力: PixInsight で開かれた画像（キャリブレーション済みが望ましい）

1. メタデータ取得
   - FITS ヘッダーから EXPTIME, PIXSCALE, GAIN, FILTER, XBINNING を読み込み
   - 取得できない項目はユーザーが手動入力

2. バックグラウンド測定
   - 全体またはユーザー指定領域でシグマクリッピング統計を実行
   - ADU_sky（中央値）を取得

3. ゼロ点取得
   - FITSヘッダー（MAGZERO 等）から自動取得、またはユーザー手動入力
   - オプション: 画像内の星をカタログと照合して実測ゼロ点を算出

4. SQM 算出
   - μ = -2.5 × log₁₀(ADU_sky / t_exp) + ZP + 5 × log₁₀(pixel_scale)

5. 結果表示・エクスポート
   - Dialog に SQM 値・各パラメータを表示
   - CSV / テキストへのエクスポート
```

## 8. 誤差要因と精度評価

### 8.1 各パラメータの不確かさ

| パラメータ | 典型的な不確かさ | SQM への影響 |
|-----------|----------------|-------------|
| ゼロ点 ZP | ±0.2〜0.5 mag（手動入力時） | 直接 ±0.2〜0.5 mag/arcsec² |
| ピクセルスケール | ±1〜2%（WCS未使用時） | ±0.02〜0.04 mag/arcsec²（小さい） |
| バックグラウンド ADU | ±数 ADU（星除去の精度に依存） | フィールドに多くの星があると増大 |
| 露出時間 | ±0.01 秒程度 | 無視できる |
| 大気消光 | 0.1〜0.3 mag/気団（天頂角に依存） | 低高度の天体では大きな誤差源 |

### 8.2 精度向上のための推奨事項

- **ゼロ点はカタログ較正で実測する**: 手動入力より大幅に精度向上（誤差 ±0.05〜0.1 mag 程度）
- **プレートソルブ済み画像を使用**: 高精度なピクセルスケールを WCS から取得可能
- **天頂付近を測定**: 大気消光の影響が最小（天頂角 < 30° 推奨）
- **キャリブレーション済み画像を使用**: バイアス・ダーク・フラット補正済みの画像が必須
- **星の多い領域を避ける**: バックグラウンド推定の精度が低下する

### 8.3 SQM デバイスとの比較

画像から算出した SQM 値と Unihedron SQM デバイスの値は、以下の理由から厳密には一致しません。

- **分光感度の違い**: SQM デバイスは V バンド相当。撮影フィルターによって異なる
- **視野の違い**: SQM デバイスは 80° 広視野平均。画像は特定の方向の狭い視野
- **大気消光**: 測定方向の大気質量によって補正量が異なる

それでも同一フィルター・同一方向での相対比較（遠征地の評価、時間変化の追跡）には十分な精度が得られます。
