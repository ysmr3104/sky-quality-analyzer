## 1. SQMとは何か

### 1.1 空の明るさの指標としての SQM

SQM（Sky Quality Meter）は、夜空の明るさを定量的に表す指標です。Unihedron 社が開発したハンドヘルドデバイス「Sky Quality Meter」に由来しており、現在では測定値そのものを指す用語として広く使われています。

SQM 値の単位は **mag/arcsec²（等級/角秒²）** です。天文学の「等級（magnitude）」は対数スケールで定義されているため、**SQM 値が大きい（＝数値が大きい）ほど空が暗い**という、直感とは逆の関係になります。

```
SQM 22.0  →  暗い空（良好）
SQM 18.0  →  明るい空（光害あり）
```

### 1.2 単位 mag/arcsec² の意味

`mag/arcsec²` は「1角秒²あたりの面輝度を等級で表したもの」です。

- **mag（等級）**: 天体の明るさを表す対数スケール。5等級の差がフラックス比 100 倍に相当
- **arcsec²（角秒²）**: 天球上の面積の単位。1角秒 = 1/3600 度 = 約 4.85 × 10⁻⁶ ラジアン

つまり SQM 値とは、「夜空の 1角秒²の明るさが何等級か」を表します。空全体を均一な面光源とみなし、その面輝度を測定しています。

### 1.3 典型的な値と空の状態

| SQM 値（mag/arcsec²） | 空の状態 |
|----------------------|---------|
| 22.0 以上             | 完璧な暗黒空。天の川の中心部が地上に影を作るほど |
| 21.5 〜 22.0          | 真の暗黒空。微光天体の観測が可能 |
| 21.0 〜 21.5          | 優良。農村地帯の空 |
| 20.0 〜 21.0          | 良好。郊外の空 |
| 19.0 〜 20.0          | 普通。郊外と都市の中間 |
| 18.0 〜 19.0          | 光害あり。明るい郊外 |
| 18.0 未満             | 都市部。天の川はほぼ見えない |

### 1.4 ボルトル暗天スケールとの対応

ボルトル（Bortle）暗天スケールは、夜空の暗さを 1〜9 のクラスで表す指標です。SQM 値との対応はおおよそ以下のとおりです。

| ボルトルクラス | SQM（mag/arcsec²） | 特徴 |
|-------------|-------------------|------|
| 1           | 22.0 以上          | 完璧な暗黒空 |
| 2           | 21.5 〜 22.0       | 真の暗黒空 |
| 3           | 21.3 〜 21.5       | 農村の空 |
| 4           | 20.4 〜 21.3       | 農村・郊外の移行帯 |
| 5           | 19.1 〜 20.4       | 郊外の空 |
| 6           | 18.0 〜 19.1       | 明るい郊外 |
| 7           | 17.5 〜 18.0       | 郊外・都市の移行帯 |
| 8 〜 9      | 17.5 未満          | 都市部 |

ボルトルスケールは観測者の主観も含むため、SQM 値のほうが定量的・再現性のある指標として科学的に有用です。

### 1.5 なぜ天文観測にとって重要か

SQM 値は以下の用途で活用されます。

- **撮影条件の記録**: 同じ機材・対象でも、SQM 値が異なれば比較できない。撮影ログに SQM を残すことで客観的な条件管理が可能
- **画像処理の基準**: バックグラウンドのノイズ特性は SQM に依存する。処理パラメータの参考値として利用
- **遠征先の評価**: 候補地の空質を定量的に比較・蓄積できる
- **光害マップへの貢献**: 個人の観測データを Globe at Night などのプロジェクトに提供し、光害の長期変化を追跡

## 2. 夜空の輝度の構成要素

SQM が測定するのは「夜空全体の明るさ」です。これは複数の異なる発光・散乱成分の合計です。各成分を理解することは、SQM 値を正しく解釈するうえで重要です。

### 2.1 光害（人工光の散乱）

都市・住宅・道路の照明が大気中の分子・エアロゾルで散乱したもの。SQM に最も大きな影響を与える成分です。

- 波長は光源の種類に依存（LED 照明が増加傾向）
- 大気の透明度（湿度・塵）によって散乱量が変化
- 地平線付近ほど強く、天頂方向は相対的に弱い

### 2.2 大気光（Airglow）

地球大気上層（高度 85 〜 300 km）での化学発光。光害のない場所でも SQM の下限を決める自然の成分です。

- 主な発光: OI（557.7 nm の緑、630 nm の赤）、OH バンド（近赤外）
- 天頂 SQM に対して約 1〜2 mag/arcsec² 分の影響
- 太陽活動サイクルや季節によって変化

### 2.3 黄道光・対日照

太陽系内の塵が太陽光を散乱したもの。春の夕方や秋の明け方に黄道に沿って見られる淡い光です。

- 黄道面に沿って広がる（黄道光）、地球の正反対方向に集中（対日照）
- 天頂 SQM への影響は小さいが、低緯度の観測地では考慮が必要

### 2.4 天の川・銀河背景光

分解できない無数の恒星が重なった拡散光。視線方向（銀緯）によって大きく異なります。

- 銀河中心方向（いて座付近）では最も明るく、SQM を数 tenths mag/arcsec² 低下させる
- 銀緯が高い方向では影響が小さい

### 2.5 まとめ

画像から求めた SQM 値は、これらすべての成分を含む **「その方向の空の合計面輝度」** です。「光害のない場所でも SQM が 22.0 を大きく超えない」のは、大気光・黄道光・銀河背景光が自然の下限を作っているためです。

## 3. 従来の SQM 測定方法

### 3.1 Unihedron SQM デバイスの仕組み

Unihedron 社の Sky Quality Meter は、光-周波数変換センサー（TAOS TSL237S）と Hoya CM-500 レンズを組み合わせたデバイスです。

- **視野角**: 約 80° FWHM（広角）
- **分光感度**: ピーク約 500〜600 nm（視感度に近い V バンド相当）
- **測定範囲**: 10 〜 22 mag/arcsec²
- **精度**: ±0.1 mag/arcsec²

センサーに入射した光量を周波数（Hz）に変換し、ファームウェアが SQM 値に変換して出力します。SQM-L（レンズ付き、狭視野版）と SQM-LE（イーサネット接続版）もあります。

### 3.2 デバイス測定の限界

| 制約 | 内容 |
|------|------|
| 広視野平均 | 80° 視野内の平均値のため、局所的な空の情報は得られない |
| 分光情報なし | 単一バンドの測定のため、光害の種類（LED vs. 水銀灯）を区別できない |
| 絶対較正 | 個体差・経年変化による誤差が生じる場合がある |
| 記録の手間 | 撮影と同時に自動記録するには追加のハードウェアが必要 |

### 3.3 画像から算出するアプローチとの比較

| 項目 | SQM デバイス | 画像から算出（本ツール） |
|------|-------------|------------------------|
| 追加ハードウェア | 必要 | 不要（撮影画像のみ） |
| 視野 | 広視野平均（80°） | 撮影視野内（フィルター・方向特定可能） |
| 分光情報 | なし（V バンド相当のみ） | フィルターに対応 |
| 時刻記録 | 自動 | FITS ヘッダー（DATE-OBS）から取得 |
| 精度 | ±0.1 mag/arcsec²（高精度） | ゼロ点較正の質に依存（±0.2〜0.5 程度） |
| 後処理での利用 | できない（リアルタイム測定のみ） | 既存の撮影データから後処理で算出可能 |

画像からの算出はデバイスより精度が劣る場合がありますが、**追加機材なしに既存の撮影データを活用できる**点が最大のメリットです。

## 4. 画像から SQM を算出する原理

### 4.1 測光の基礎

天文測光では、天体の明るさを「等級（magnitude）」で表します。等級の定義は以下のとおりです。

```
m = -2.5 × log₁₀(F) + C
```

- `F`: 検出器に入射したフラックス（光子数/秒 など）
- `C`: ゼロ点定数（測光系によって決まる）

等級は **対数スケール** であり、5等級の差がフラックス比 100 倍に対応します。また、**等級が小さいほど明るい**（負の値ほど非常に明るい）という点に注意が必要です。

### 4.2 面輝度（mag/arcsec²）の概念

点源（星）の明るさは「等級」で表しますが、空のバックグラウンドや系外銀河など広がった天体の明るさは **面輝度（Surface Brightness）** で表します。

面輝度は「単位立体角あたりの明るさを等級で表したもの」であり、単位は mag/arcsec²（等級/角秒²）です。

等級と面輝度の関係:
```
μ = m_pixel + 2.5 × log₁₀(Ω)
```

- `μ`: 面輝度 [mag/arcsec²]
- `m_pixel`: 1ピクセルの等級
- `Ω`: 1ピクセルがカバーする立体角 [arcsec²]

1ピクセルが `s` arcsec/px のピクセルスケールを持つ場合、`Ω = s²` [arcsec²/px] です。

### 4.3 バックグラウンド輝度の測定

撮影画像のピクセル値（ADU）には、以下の成分が含まれています。

```
ADU_total = ADU_sky + ADU_bias + ADU_dark + ADU_stars
```

- `ADU_sky`: 空のバックグラウンド（求めたい成分）
- `ADU_bias`: バイアス（読み出しオフセット）
- `ADU_dark`: ダーク電流
- `ADU_stars`: 星からの寄与

キャリブレーション済みの画像（バイアス・ダーク・フラット補正済み）では、バイアスとダーク電流は除去されています。さらに星を除外した領域の中央値または平均値を取ることで、`ADU_sky` を推定します。

### 4.4 ADU から面輝度への変換の考え方

キャリブレーション済み画像の空バックグラウンド `ADU_sky` [ADU/px] から面輝度（SQM 値）を算出する流れ:

1. **ADU を電子数に変換**: `e⁻ = ADU × Gain` [e⁻/px]
2. **露出時間で割る**: `flux = e⁻ / t_exp` [e⁻/s/px]
3. **等級に変換（ゼロ点使用）**: `m_pixel = -2.5 × log₁₀(flux) + ZP`
4. **ピクセルスケールで面輝度に変換**: `μ = m_pixel + 2.5 × log₁₀(pixel_scale²)`

この `μ` が SQM 値に相当する面輝度です。

## 5. 算出に必要なパラメータ

### 5.1 パラメータ一覧

| パラメータ | 単位 | 取得元 | 説明 |
|-----------|------|--------|------|
| `ADU_sky` | ADU/px | 画像解析 | バックグラウンド輝度（中央値） |
| `t_exp` | 秒 | FITS: `EXPTIME` | 露出時間 |
| `pixel_scale` | arcsec/px | FITS: `PIXSCALE` または計算 | ピクセルスケール |
| `Gain` | e⁻/ADU | FITS: `GAIN` またはカメラ仕様 | カメラゲイン |
| `ZP` | mag | 測光較正 | ゼロ点等級 |
| `Filter` | — | FITS: `FILTER` | 使用フィルター |
| `Binning` | — | FITS: `XBINNING` | ビニング（ピクセルスケール補正に必要） |

### 5.2 露出時間（EXPTIME）

FITS ヘッダーの `EXPTIME` または `EXPOSURE` キーワードから取得します。単位は秒。露出時間が長いほど同じ空でも ADU が大きくなるため、正規化に必須のパラメータです。

### 5.3 ピクセルスケール（pixel_scale）

ピクセルスケールは 1 ピクセルが対応する天球上の角度（arcsec/px）です。

FITS ヘッダーに `PIXSCALE` がある場合はそれを使用します。ない場合は以下の式で計算できます。

```
pixel_scale = (pixel_size_μm / focal_length_mm) × 206.265
```

- `pixel_size_μm`: カメラのピクセルサイズ [μm]
- `focal_length_mm`: 鏡筒の焦点距離 [mm]
- `206.265`: 変換係数（ラジアン→角秒）

**ASI294MC Pro + RedCat 51 の場合:**

```
pixel_scale = (4.63 μm / 250 mm) × 206.265 ≈ 3.82 arcsec/px
```

この広いピクセルスケールは SQM 算出に有利です（1ピクセルが広い空の面積を捉えるため、バックグラウンド統計が安定しやすい）。

プレートソルブ済みの画像では WCS キーワード（`CD1_1`, `CD1_2` など）から高精度なピクセルスケールを導出できます。

### 5.4 ゲイン（Gain）

ゲインは 1 ADU が何電子（e⁻）に対応するかを表す変換係数です（単位: e⁻/ADU）。

**ZWO カメラ（ASI294MC Pro 等）の FITS ヘッダーの注意点:**

ZWO カメラの撮影ソフト（N.I.N.A., SharpCap, APT 等）が書き込む `GAIN` キーワードは、**カメラのゲイン設定値（0〜570 の整数）** であり、e⁻/ADU の変換係数ではありません。実際の e⁻/ADU は `EGAIN` キーワードに書き込まれる場合があります（ソフトウェアによる）。

ASI294MC Pro のゲイン設定値と実効変換係数の対応:

| ゲイン設定値 | モード | e⁻/ADU | 読み出しノイズ |
|------------|--------|--------|-------------|
| 0          | LCG    | ~3.6   | ~7.4 e⁻    |
| 120        | HCG    | ~0.97  | ~1.2 e⁻    |
| 200        | HCG    | ~0.49  | ~1.2 e⁻    |

HCG（High Conversion Gain）モードはゲイン設定値 120 付近から切り替わり、読み出しノイズが大幅に低減されます。天体撮影では通常 HCG モード（ゲイン 120 以上）が使用されます。

ゲインが不明な場合や不確かな場合は、ゼロ点較正を ADU ベースで行うことでゲインを陽に使わずに算出できます（後述）。

### 5.5 ゼロ点等級（ZP）

ゼロ点（Zero Point）は測光系を絶対等級スケールに結びつける定数です。

```
m_true = -2.5 × log₁₀(ADU / t_exp) + ZP
```

ゼロ点の求め方（精度の高い順）:

1. **カタログ較正**: 画像内の既知の星のカタログ等級と測定値を比較して算出（最も精度が高い）
2. **Astrometry.net / Siril の出力**: プレートソルブや測光較正を行ったソフトウェアが FITS ヘッダーに書き込む場合がある（`MAGZERO`, `MAGZP` など）
3. **経験値**: 機材・フィルター・大気条件の典型値を手動入力（最も簡便だが精度は低い）

### 5.6 フィルター

使用フィルターによって測定する波長域が異なるため、SQM 値の解釈が変わります。

| フィルター | 中心波長 | SQM との関係 |
|-----------|---------|-------------|
| L（ルミナンス）| 全可視域 | Unihedron SQM に近い（V バンド相当） |
| V（緑） | 550 nm | SQM デバイスの分光感度に最も近い |
| R（赤） | 640 nm | 光害（LED・ナトリウム灯）に敏感 |
| Ha（Hα） | 656 nm | 輝線フィルター。光害の影響を大幅にカット |
| B（青） | 450 nm | 大気散乱の影響が大きい |

フィルターなし（OSC/DSLR の RGB 合成）の場合は実効波長が不明確になるため、別途考慮が必要です。

**ASI294MC Pro（フィルターなし）の場合:**

OSC カメラはフィルターなしでも RGGB ベイヤー配列により各ピクセルが R/G/B のいずれかの色を受光します。SQM 算出には **グリーンチャンネルを使用する** のが最適です。

理由:
- IMX294 の G チャンネルのピーク感度は約 520〜540 nm で、Unihedron SQM デバイス（ピーク約 500〜600 nm）の分光感度に最も近い
- RGGB 配列ではグリーンピクセルが全体の 50% を占めるため、統計的に最も安定

PixInsight での G チャンネル取得方法:
- デベイヤー後の RGB 画像: `ChannelExtraction` プロセスで G チャンネルを抽出
- デベイヤー前の CFA 画像: `CFAExtract` で G ピクセルのみを抽出

## 6. 推奨撮影手順

SQM 値を精度よく算出するためには、撮影段階でのセットアップが重要です。本章では基準星方式（7章参照）を前提とした撮影手順を説明します。

### 6.1 撮影の基本方針

測定の本質は「**画像内の基準星（等級既知）と空のバックグラウンドを同じ条件で記録し、両者のカウント数を比較する**」ことです。そのため以下の条件がすべてのフレームで統一されている必要があります。

- ゲイン設定値
- 冷却温度
- ビニング
- ピント位置（意図的にボカした位置で固定）

### 6.2 ピントをボカす理由と方法

**最重要ポイント**です。SQM 測定用の撮影では、**ピントを意図的にやや甘くして撮影します**。

**理由:** ピントが合っていると明るい基準星の光が 1〜数ピクセルに集中し、センサーの最大値（ASI294MC Pro では 65535 ADU）に達して飽和（サチる）してしまいます。飽和したピクセルは正しいカウント値を失うため、測光値が不正確になります。

**方法:** ピントリングをわずかにずらし、基準星の像が直径 10〜20 ピクセル程度の円形に広がる状態にします。背景の均一性には影響しないため、バックグラウンド測定精度は変わりません。

```
ピント合わせ時:  ●  (1〜3px に集中 → 飽和のリスク)
ボカした状態:   ○  (10〜20px に広がる → 飽和しない)
```

### 6.3 露光時間と撮影枚数

複数の露光時間で各 1 枚ずつ撮影します。

| コマ | 露光時間 |
|-----|---------|
| 1   | 1 秒    |
| 2   | 2 秒    |
| 3   | 4 秒    |
| 4   | 6 秒    |
| 5   | 8 秒    |
| 6   | 10 秒   |

**複数枚撮影する理由:**

カウント値が露光時間に比例して線形に増加しているかを確認するためです。

- グラフが直線になっていれば正常（飽和・非線形性なし）
- 直線から外れるコマがある場合は、雲の通過・飽和・その他の異常を疑う

計算には傾き（= カウント値 / 露光時間）を使うため、複数点から最小二乗フィットで傾きを求めることで誤差を低減できます。

**ASI294MC Pro + RedCat 51 での目安:**

| ゲイン設定 | 推奨最長露光時間 | 根拠 |
|-----------|----------------|------|
| 120（HCG）| 10〜30 秒      | 基準星 5〜6 等なら 10 秒程度で十分なカウントを確保できる |

明るい基準星（3〜4 等）を使う場合は最長露光時間を短めに設定してください。

### 6.4 ダーク補正

各露光時間と同じ条件（同ゲイン・同温度・同露光時間）でダークフレームを撮影します。ダーク補正を行うことで、ダーク電流とバイアスオフセットを除去できます。

```
補正後 ADU = 撮影フレーム ADU − ダークフレーム ADU
```

### 6.5 基準星の選び方

基準星は以下の条件を満たす恒星を選びます。

| 条件 | 推奨値 | 理由 |
|------|--------|------|
| 等級 | **V < 4等**（明るい空では V < 2等が理想） | 暗すぎると空のノイズに埋もれる。必要等級はサイトの空の明るさに依存する（後述） |
| 位置 | 画像の中央付近 | 周辺減光（ビネット）の影響を避ける |
| 周辺 | 近傍に同等輝度の星がない | 開口測光の際に隣接星が混入しないように |
| カタログ | Hipparcos / Tycho-2 など精度の高い V 等級が既知 | SQM の精度に直結 |

**空の明るさと必要等級の目安（アパーチャ r=15px, ピクセルスケール 3.82"/px の場合）:**

| サイトの SQM | 推奨等級 | 理由 |
|------------|---------|------|
| 18〜22（暗い空） | V < 7 | 背景ノイズが小さいため比較的暗い星でも測定可能 |
| 16〜18（郊外） | V < 5 | 背景ノイズが大きくなるため明るい星が必要 |
| < 16（都市部・明るい郊外） | V < 3 | V=7〜8 程度の星はノイズに完全に埋もれる |

> **注意:** RedCat 51（画角 4.4° × 2.9°）でも、都市部のような明るいサイトでは V < 3 の星を画角内に収める必要があり、方向の選択が重要になります。撮影前に SIMBAD 等で明るい基準星（α UMa: V=1.8、β UMa: V=2.4 など）の位置を確認してください。

### 6.6 撮影方向

SQM 測定の目的に応じて撮影方向を選択します。

| 目的 | 推奨方向 |
|------|---------|
| 光害評価（標準的な測定） | 天頂（天頂角 < 30°）。大気消光の影響が最小 |
| 特定方向の光害 | 問題の方向に向けて測定 |
| Unihedron SQM との比較 | 天頂方向。SQM デバイスは通常天頂向けで測定 |

### 6.7 撮影チェックリスト

```
□ ゲイン・冷却温度・ビニングを設定し記録する
□ ピントを意図的にボカした位置で固定する
□ 基準星が画角内に収まる方向を選ぶ
□ 露光時間 1, 2, 4, 6, 8, 10 秒で各 1 枚撮影する
□ 同条件でダークフレームを撮影する（各露光時間ごと、または最長露光のみ）
□ 全フレームでカメラ設定が変わっていないことを確認する
```

## 7. SQM 算出の数学

### 7.1 バックグラウンド推定アルゴリズム

天文画像のバックグラウンドには星・銀河などの点源・拡散源が含まれるため、単純な平均では過大評価になります。以下のアルゴリズムで空のバックグラウンドレベル `ADU_sky` を推定します。

**シグマクリッピング統計**:

1. ユーザーが指定した星のない **64×64 px の ROI（領域）** のピクセルを収集
2. 中央値（median）と標準偏差（σ）を計算
3. 中央値 ± n×σ（通常 n=3、5回反復）の範囲外の値を除外
4. 残ったピクセルの中央値を `ADU_sky` とする

中央値を使う理由: 平均値は輝点（星）の影響を強く受けるが、中央値は外れ値に頑健です。

ROI を使う理由: 画像全体を使うと銀河・星雲・明るい星が多い領域で過大評価になるため、ユーザーが星のない均一な空領域を明示的に指定します。

### 7.2 SQM 算出の 2 つのアプローチ

画像から SQM 値を算出する方法には大きく 2 つのアプローチがあります。

---

#### アプローチ A: ゼロ点方式

測光ゼロ点（ZP）を用いて背景の面輝度を直接算出する方法です。

**計算式:**

```
μ = -2.5 × log₁₀(ADU_sky / t_exp) + ZP_ADU + 5 × log₁₀(pixel_scale)
```

**式の導出:**

```
1. 背景の単位時間あたりのフラックス [ADU/s/px]:
   f_sky = ADU_sky / t_exp

2. ピクセル等級（ゼロ点で絶対等級スケールに変換）:
   m_pixel = -2.5 × log₁₀(f_sky) + ZP_ADU

3. 1 ピクセルがカバーする立体角 [arcsec²]:
   Ω = pixel_scale²

4. 面輝度に変換:
   μ = m_pixel + 2.5 × log₁₀(Ω)
     = m_pixel + 5 × log₁₀(pixel_scale)

5. まとめ:
   μ = -2.5 × log₁₀(ADU_sky / t_exp) + ZP_ADU + 5 × log₁₀(pixel_scale)
```

**ゼロ点の求め方:**

```
ZP_ADU = m_catalog + 2.5 × log₁₀(ADU_star / t_exp)
```

画像内の既知等級の星を測光し、その実測 ADU とカタログ等級からゼロ点を算出します。複数の星の中央値をゼロ点とします。

**長所:**
- 撮影済みの任意の画像に適用できる
- ゼロ点さえ求まれば基準星が画角外でもよい

**短所:**
- ゼロ点の精度が SQM 値の精度に直結する
- カタログ照合のための星同定が必要

---

#### アプローチ B: 基準星方式（本ツールで採用）

画像内の等級既知の基準星と空のバックグラウンドの **比** からSQM値を算出する方法です。ゼロ点を陽に求めることなく、基準星自体が較正器の役割を果たします。

**計算式:**

```
SQM = 2.5 × log₁₀(L_star / L'_sky) + m₀
```

| 変数 | 意味 | 単位 |
|------|------|------|
| `L_star` | 基準星の単位時間あたりカウント数 | counts/s |
| `L'_sky` | 背景の単位面積・単位時間あたりカウント数 | counts/arcsec²/s |
| `m₀` | 基準星のカタログ等級 | mag |

**各変数の求め方:**

`L_star` は複数の露光時間で測光した基準星のカウント値と露光時間のグラフの傾きです:

```
L_star = slope of (ADU_star vs t_exp)   [counts/s]
```

`L'_sky` は背景カウント値を露光時間とピクセルスケールで正規化したものです:

```
L'_sky = L_sky / θ_pixel²

ここで:
  L_sky    = slope of (ADU_sky vs t_exp)   [counts/s/px]
  θ_pixel  = pixel_scale                   [arcsec/px]
```

**式の導出:**

```
基準星の等級を m₀ とすると、等級の定義から:
  m₀ = -2.5 × log₁₀(L_star) + ZP   … (1)

背景の面輝度 SQM は:
  SQM = -2.5 × log₁₀(L'_sky) + ZP  … (2)

(1) - (2) で ZP を消去:
  m₀ - SQM = -2.5 × log₁₀(L_star) + 2.5 × log₁₀(L'_sky)
            = 2.5 × log₁₀(L'_sky / L_star)

したがって:
  SQM = m₀ + 2.5 × log₁₀(L_star / L'_sky)
```

**長所:**
- ゼロ点の算出が不要（ZP が式中でキャンセルされる）
- ゲインや絶対的な測光較正が不要
- 同一画像内の相対測光のみで完結するため大気消光・感度の絶対値の影響を受けにくい
- 参考文献の手法と一致しており、実績がある

**短所:**
- 画角内に適切な基準星（等級既知、非飽和）が必要
- 基準星の測光精度が SQM 値の精度に直結する

---

#### 両アプローチの比較

| 項目 | ゼロ点方式（A） | 基準星方式（B） |
|------|--------------|--------------|
| ゼロ点の計算 | 必要 | 不要（式中でキャンセル） |
| 基準星（画角内） | 不要 | 必要 |
| ゲイン情報 | 必要（または ZP に吸収） | 不要 |
| 大気消光の影響 | あり（天頂角補正が必要） | 相対測光のため小さい |
| 実装の複雑さ | やや複雑 | シンプル |

**本ツールではアプローチ B（基準星方式）を採用します。** ゼロ点較正不要でシンプルに実装でき、参考文献の手法とも一致しているためです。ゼロ点方式（A）は将来の拡張として実装を検討します。

### 7.3 ビニングの補正

ビニング `n × n` で撮影した画像では、1ピクセルが n² 個の元ピクセルに相当します。

```
pixel_scale_binned = pixel_scale_1x1 × n
```

FITS ヘッダーの `XBINNING` / `YBINNING` から取得し、ピクセルスケールに乗算して補正します。

## 8. 実装仕様

### 8.1 設計方針の確定事項

| 項目 | 決定事項 | 備考 |
|------|---------|------|
| 算出手法 | 基準星方式（アプローチ B） | ゼロ点不要、相対測光で完結 |
| 撮影モード | マルチ露光モード | 複数露光時間フレームから傾きを算出 |
| 入力画像 | WBPP の `calibrated/` または `debayered/` 個別フレーム | スタック前の1フレームずつ |
| プレートソルブ | 任意（あれば活用） | SQM 算出には不要。ただしプレートソルブ済み XISF があれば SIMBAD カタログ検索でクリック位置付近の明るい星を自動提案できる |
| 基準星選択 | 画像クリック → SIMBAD 座標検索 または Sesame 名前検索 | クリック位置の RA/Dec（WCS 必要）から半径 120' 以内の V<9 星を一覧表示。WCS なしの場合は手動で星名と等級を入力 |
| OSC 対応 | デベイヤー後 RGB（G チャンネル抽出）+ モノクロ 1ch | CFA（デベイヤー前）は将来対応 |
| 等級取得 | SIMBAD TAP 検索（座標ベース）または Sesame 検索（星名ベース） | 両方に対応。SIMBAD は画像内フィルタリング・Suitability 評価つき |

### 8.2 アルゴリズム概要

```
入力: WBPP の calibrated/ または debayered/ ディレクトリの個別フレーム複数枚
      （異なる露光時間・同一カメラ/鏡筒/ゲイン設定）

1. 起動・初期化
   - equipment.json を読み込み
   - 最初のフレームの INSTRUME キーワードと照合してカメラを自動選択
   - INSTRUME がない/一致しない場合はユーザーがドロップダウンから選択

2. フレーム読み込み（マルチ露光モード）
   - ユーザーがファイルダイアログで複数フレームを選択
   - 各フレームから EXPTIME を取得
   - OSC カメラ（NAXIS3 == 3）: G チャンネルを抽出
   - PixInsight 正規化値（0〜1）→ ADU への変換
     ADU = normalized × 65535（16bit）または normalized × 4294967295（32bit）

3. バックグラウンド測定 → L_sky
   - ユーザーが星のない 64×64px ROI をプレビュー画像上でクリックして指定
   - 各フレームでその ROI にシグマクリッピング統計（σ=3、5回反復）
   - ADU_sky（シグマクリッピング後の中央値）を取得
   - 全フレームの（t_exp, ADU_sky）ペアを最小二乗線形フィット
   - 傾き = L_sky [counts/s/px]、R² で線形性を確認

4. 基準星選択・測光 → L_star
   - 最長露光フレームをプレビュー表示（ScrollBox、ズーム/パン対応）
   - ユーザーが基準星をクリック → ピクセル座標を記録
   - 等級 m₀ の取得方法（いずれか）:
     a. **SIMBAD 座標検索**（プレートソルブ済み XISF の場合）: WCS でクリック位置を RA/Dec に変換 → SIMBAD TAP で半径 120' 以内の V<9 星を検索 → 画像範囲外の星を除外 → NearbyStarDialog で一覧表示・選択（Suitability 評価列つき）
     b. **Sesame 名前検索**: 星名を手動入力 → Sesame で RA/Dec と V 等級を取得
   - 各フレームの同座標で開口測光（アパーチャ半径はユーザー指定）
     - プレートソルブ済みフレームは WCS でフレームごとに星位置を補正
     - ADU_star = Σ(アパーチャ内ピクセル) − スカイ中央値 × ピクセル数
   - 全フレームの（t_exp, ADU_star）ペアを最小二乗線形フィット
   - 傾き = L_star [counts/s]、R² で線形性を確認

5. SQM 算出
   - L'_sky = L_sky / pixel_scale²   [counts/arcsec²/s]
   - SQM = 2.5 × log₁₀(L_star / L'_sky) + m₀

6. 結果表示・エクスポート
   - SQM 値・各中間値・R² を Dialog に表示
   - CSV エクスポート
```

### 8.3 PixInsight 画像値の正規化解除

PixInsight は内部的に全画像値を 0〜1 に正規化して保持します。SQM 算出には実際の ADU 値が必要なため、以下の変換を行います。

```javascript
// FITS ヘッダーの BITPIX または画像の bitsPerSample から bit depth を判定
var maxADU = (image.bitsPerSample == 16) ? 65535 : 4294967295;
var adu = image.sample(x, y, channel) * maxADU;
```

キャリブレーション済みの 32bit float 画像では、PixInsight が FITS の BZERO/BSCALE を適用済みのため、上記の変換で正しい ADU 値が得られます。

### 8.4 開口測光の実装

基準星の ADU 合計（L_star の元データ）を開口測光で取得します。

```
アパーチャ（内円）半径 r_ap  : 星の像が収まる半径（デフォルト 15px、ユーザー変更可）
スカイ内径         r_sky_in : r_ap + 5px（デフォルト 20px）
スカイ外径         r_sky_out: r_ap + 15px（デフォルト 30px）  ← スカイ環幅 = 10px

スカイ中央値 = スカイ環内ピクセルの中央値（シグマクリッピング済み）
ADU_star = Σ(アパーチャ内ピクセル) − スカイ中央値 × アパーチャ内ピクセル数
```

ピントをボカして撮影するため（6.2節参照）、アパーチャ半径を大きめに設定します。

### 8.5 SIMBAD カタログ検索と Suitability 評価

プレートソルブ済み XISF フレームが読み込まれている場合、「Find in Catalog...」ボタンで SIMBAD TAP サービスを検索し、画角内の明るい星を一覧表示します。

**検索仕様:**

| 項目 | 設定 |
|------|------|
| エンドポイント | `http://simbad.u-strasbg.fr/simbad/sim-tap/sync` (ADQL) |
| 検索半径 | 120'（プライマリ）、V<9 フィルター |
| フォールバック | V<11、半径 2° |
| 画像外フィルター | WCS 投影後に画像範囲外の星を除外 |

**Suitability（適合度）評価:**

各星に対し、現在のアパーチャとピクセルスケールを用いて SNR 比を推定します。

```
SNR比 = 10^(0.4 × (16 - V)) / (π × (r_ap × pixel_scale)²)
```

（SQM=16 の空を仮定した推定値。実際のサイト SQM により変動します）

| ラベル | SNR 比 | 目安の等級（r=15px, scale=3.82"/px） |
|--------|--------|--------------------------------------|
| Ideal | ≥ 20 | V < 2 |
| Good | 5〜20 | V < 3.5 |
| Marginal | 1〜5 | V < 5 |
| Too faint | < 1 | V ≥ 5 |
| Saturation risk | — | V < 1.5 |

#### 8.5.2 カメラフィールドの説明

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `name` | string | UI 表示名 |
| `instrume` | string | FITS `INSTRUME` キーワードの値（自動照合に使用） |
| `pixel_pitch` | number | ピクセルサイズ [μm] |
| `sensor_width` | number | センサー横幅 [px] |

| `sqm_channel` | string | SQM 算出に使うチャンネルの推奨値。OSC は `"G"`、モノクロは `"L"` |

#### 8.5.3 チャンネル選択のロジック

```
color_type == "color" の場合:
  → sqm_channel = "G"（デフォルト）
  → ユーザーが R / G / B から変更可能

color_type == "mono" の場合:
  → sqm_channel = "L"（チャンネル選択不要）

いずれも:
  → FITS の NAXIS3 == 1（モノクロ画像）なら強制的に "L" 扱い
  → FITS の NAXIS3 == 3（RGB 画像）なら R/G/B から選択
```

#### 8.5.4 ゲイン設定値から e⁻/ADU を解決するロジック

基準星方式ではゲイン（e⁻/ADU）は SQM 計算式に現れませんが、エクスポートデータの記録用に参照します。

```
1. FITS ヘッダーの EGAIN が存在する → その値を使用
2. FITS ヘッダーの GAIN（設定値）と gain_modes テーブルを照合
   → 設定値が gain_min ≤ GAIN ≤ gain_max のモードの egain を使用
3. いずれも取得できない → 記録欄は空欄
```

#### 8.5.5 鏡筒フィールドの説明

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `name` | string | UI 表示名 |
| `focal_length` | number | 焦点距離 [mm]（0 はカスタム入力） |
| `aperture` | number | 口径 [mm] |
| `f_ratio` | number | F 値 |

ピクセルスケールは選択されたカメラとビニング設定から自動計算:
```
pixel_scale = (pixel_pitch × binning / focal_length) × 206.265
```

## 9. 誤差要因と精度評価

### 9.1 各パラメータの不確かさ

基準星方式では ZP がキャンセルされるため、誤差の主要因はゼロ点ではなく基準星の測光精度とバックグラウンド推定精度です。

| パラメータ | 典型的な不確かさ | SQM への影響 |
|-----------|----------------|-------------|
| 基準星カタログ等級 m₀ | ±0.01〜0.05 mag（Hipparcos/Tycho-2） | 直接 ±0.01〜0.05 mag/arcsec² |
| 基準星測光（L_star） | ±1〜3%（飽和・近傍星の有無に依存） | ±0.03〜0.07 mag/arcsec² |
| バックグラウンド（L_sky） | ±数 ADU（星除去の精度に依存） | 銀河面・星雲付近では増大 |
| ピクセルスケール | ±1〜2%（機材 DB からの計算時） | ±0.02〜0.04 mag/arcsec²（小さい） |
| 線形フィットの R² | 1.0 に近いほど良好 | R² < 0.99 なら測定を疑う |
| 大気消光 | 相対測光のため小さい | 基準星と背景が同一大気経路のためほぼキャンセル |

### 9.2 精度向上のための推奨事項

- **基準星は Hipparcos / Tycho-2 の V 等級が既知の星を使う**: カタログ等級の精度が SQM に直結
- **線形 R² が 0.99 以上であることを確認**: 下回る場合は雲・飽和・フレームのばらつきを疑う
- **ピントのボカし量を適切に設定**: 飽和すると R² が崩れる（6.2節参照）
- **天頂付近を撮影する**: 大気消光は相対測光でキャンセルされるが、視程の悪い低空は避ける
- **WBPP のキャリブレーション済みフレームを使う**: フラット補正により周辺減光の影響を低減
- **基準星の周辺に近傍の明るい星がない場所を選ぶ**: 開口測光への混入を防ぐ

### 9.3 SQM デバイスとの比較

画像から算出した SQM 値と Unihedron SQM デバイスの値は、以下の理由から厳密には一致しません。

- **分光感度の違い**: SQM デバイスは V バンド相当。撮影フィルターによって異なる
- **視野の違い**: SQM デバイスは 80° 広視野平均。画像は特定の方向の狭い視野
- **大気消光**: 測定方向の大気質量によって補正量が異なる

それでも同一フィルター・同一方向での相対比較（遠征地の評価、時間変化の追跡）には十分な精度が得られます。
